<Section icon={Cpu} title="動作環境と携帯機でのパフォーマンス">

**Dear Passengers**がSteam Deckでどの程度動作するかを理解するために、公式の最小動作環境を確認してみましょう。本作は比較的軽量に設計されており、デスクトップPC向けにはGTX 1060またはRX 6600 XTのみを要求しています。Steam DeckのGPUは800p解像度においてこれらと同等のパフォーマンスを発揮できるため、動作の見通しは非常に明るいです。

携帯機プレイヤーにとって最大の懸念材料となるのは、CPU負荷の高い物理演算です。乱気流が発生すると、すべての乗客、機内食のトレイ、荷物が物理演算オブジェクトへと変化します。これにより、マナーの悪い乗客や固定されていない荷物で機内が溢れかえる「極限（Extreme）」リスクのフライトでは、フレームレートが低下する可能性があります。

| コンポーネント | PC最小動作環境 | Steam Deckのスペック |
| :--- | :--- | :--- |
| **プロセッサー** | Intel Core i5 @ 2.5 GHz | カスタムZen 2 (4コア/8スレッド) |
| **メモリ** | 8 GB RAM | 16 GB LPDDR5 |
| **グラフィックス** | GTX 1060 / RX 6600 XT | カスタムRDNA 2 (8 CUs) |
| **DirectX** | Version 12 | 対応 |
| **ストレージ** | 4 GB | 高速NVMe SSD / microSD |

<Callout type="tip" title="最適化のコツ">
最も安定した動作を得るには、Steam Deckのリフレッシュレートを**40Hz/40FPS**にロックすることをお勧めします。これにより、30FPSよりも滑らかな映像を維持しつつ、長時間のフライトでもバッテリー消費を抑えることができます。
</Callout>

<Section icon={AlertTriangle} title="Steam Deckで機内ハザードを生き抜く方法">

このゲームの核心は、リスクとリターンの管理にあります。すべてのフライトの前に、積み込む貨物や乗客の難易度を選択します。報酬が高いフライトほど、密輸品やシートベルト着用指示に従わない問題のある乗客といった「極限（Extreme）」リスクが伴います。Steam Deckにおいて、これらの混沌としたシーンは最も負荷が高くなります。

乱気流中、固定されていない荷物などの物理オブジェクトは凶器と化します。Steam Deckの画面は小さいため、散らかった機内は見づらくなります。通路をクリアに保つことは、単なる安全規則ではなく、描画オブジェクトを減らしてフレームレートを安定させるための「パフォーマンス最適化」でもあるのです。

| ハザードの種類 | 一次対応者 | エスカレーションリスク |
| :--- | :--- | :--- |
| **ダイナミックな天候** | パイロット | 客室全体に物理的な大混乱を引き起こす |
| **バードストライク** | パイロット / 機関士 | エンジン火災および揚力の喪失に繋がる |
| **貨物の脱走** | 客室乗務員 | 乗客に危害を加え、通路を塞ぐ |
| **機内火災** | 最寄りのクルー | 貨物室やコックピットへ急速に延焼する |

<Callout type="warning" title="物理演算に関する警告">
物理オブジェクトが多発する緊急事態では、一時的にフレームタイムのスパイク（カクつき）が発生する可能性があります。バッテリー残量が少ない場合や、本体が熱を帯びている状態での「極限（Extreme）」リスクフライトは避けることをお勧めします。
</Callout>
