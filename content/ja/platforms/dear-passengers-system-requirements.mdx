<Section icon={Shield} title="Dear Passengers システム要件：PCスペック">

**Dear Passengers システム要件** を満たすことで、詳細な環境のレンダリングや高速な物理演算をシステムが処理できるようになります。このゲームは幅広いハードウェアで動作するように設計されていますが、推奨スペックを満たしていれば、複数のプレイヤーが同時に画面上で操作しているときでもスタッタリング（カクつき）を防ぐことができます。

以下の表は、ゲームを実行するために必要な最小および推奨ハードウェアスペックをまとめたものです。

| ハードウェア構成要素 | 最小要件 | 推奨スペック |
| :--- | :--- | :--- |
| **OS** | Windows 10 (64-bit) | Windows 10/11 (64-bit) |
| **プロセッサ (CPU)** | Intel Core i3-6100 / AMD FX-6300 | Intel Core i5-8400 / AMD Ryzen 5 2600 |
| **メモリ (RAM)** | **8 GB RAM** | **16 GB RAM** |
| **グラフィックカード (GPU)** | GTX 960 / Radeon RX 470 | GTX 1060 / Radeon RX 580 |
| **DirectX バージョン** | DirectX 11 | DirectX 12 |
| **ストレージ空き容量** | 10 GB の空き容量 | 10 GB の SSD 空き容量 |

予算を抑えたいプレイヤーにとって、最小スペックは1080p解像度の低設定で安定した30 FPSを目標としています。ただし、マルチプレイロビーをホストしたり、ゲームプレイを配信したりする場合は、パフォーマンスの低下を避けるために推奨スペックへのアップグレードを強くお勧めします。

<Callout type="warning" title="システムアーキテクチャ">
32ビットオペレーティングシステムでの動作はサポートされておらず、起動時に即座にクラッシュする可能性があります。
</Callout>

<Section icon={Sparkles} title="パフォーマンスの最適化とグラフィック設定">

ゲーム内設定を微調整することで、全体のビジュアル品質を損なうことなくフレームレートを大幅に向上させることができます。このゲームは高速な移動と素早いカメラの切り替えが特徴であるため、ゲームプレイ中の出来事を追跡し、機内の緊急事態に対応するには、高いフレームレートを維持することが重要です。

個々のグラフィック設定がパフォーマンスに与える影響については、以下の表を参照してください。

| グラフィック設定 | パフォーマンスへの影響 | 低スペックPCの推奨値 | ハイエンドPCの推奨値 |
| :--- | :--- | :--- | :--- |
| **シャドウ品質** | **高い影響** | 低 / 中 | 高 |
| **アンチエイリアス** | **中程度の影響** | SMAA または オフ | テンポラルAA (TAA) |
| **テクスチャ詳細** | **低い影響** | 中（VRAMに依存） | 高 / ウルトラ |
| **ポストプロセッシング** | **高い影響** | オフ / 低 | 高 |
| **V-Sync (垂直同期)** | **変動する影響** | オフ (G-Sync/FreeSyncを使用) | オン (画面のティアリングが発生する場合) |

ゲームを起動する前に、以下の手順に従ってOSとハードウェア構成を最適化してください。

<Steps>
  <Step num="1" title="グラフィックドライバーの更新">
    NVIDIA、AMD、またはIntelから最新のドライバーをダウンロードしてインストールし、互換性を確保して特定のパフォーマンスパッチを適用します。
  </Step>
  <Step num="2" title="Windows 電源プランの調整">
    激しいゲームプレイ中にCPUのスロットリングを防ぐため、OSの電源プランを「高パフォーマンス」に設定します。
  </Step>
  <Step num="3" title="ハードウェアアクセラレータによるGPUスケジューリングの有効化">
    Windowsのグラフィック設定メニューでこの設定をオンにして、レイテンシを削減し、フレームタイムの一貫性を向上させます。
  </Step>
  <Step num="4" title="ゲーム内プリセットの設定">
    ゲームを起動し、上の表にある最適化された設定を適用します。FPSを最も向上させるためにシャドウの削減を優先してください。
  </Step>
</Steps>

<Callout type="tip" title="VRAMの管理">
グラフィックカードのVRAMが4GB未満の場合は、新しい環境を読み込む際の深刻なスタッタリングを防ぐために、テクスチャ品質を「中」以下に設定してください。
</Callout>

<Section icon={Trophy} title="ストレージとインストール前のチェックリスト">

設置面積は現代のAAAタイトルに比べて比較的小さいですが、ストレージデバイスの速度は、マッチのロード時間や遷移中のアセットストリーミングにおいて依然として重要な役割を果たします。ロード画面を最小限に抑えるために、ソリッドステートドライブ（SSD）にゲームをインストールすることを推奨します。

以下の表は、ストレージメディアのオプションによるパフォーマンスの違いを示しています。

| ストレージデバイスの種類 | 平均ロード時間 | アセットストリーミング性能 | 推奨度 |
| :--- | :--- | :--- | :--- |
| **標準HDD** | 遅い (30-45秒) | テクスチャのポップインの可能性あり | 非推奨 |
| **SATA SSD** | 速い (10-15秒) | スムーズなアセット読み込み | 推奨 |
| **NVMe M.2 SSD** | 非常に速い (5秒未満) | 瞬時のロード | 強く推奨 |

<Section icon={Swords} title="協力マルチプレイのシステム負荷">

協力マルチプレイをプレイすると、特にロビーホストを務めるプレイヤーのハードウェアに負荷がかかります。ホストのマシンは、プレイヤーの位置、物理同期データ、およびネットワークパケットを同時に処理する必要があり、CPUとメモリの使用率が増加します。

以下の表は、ホストとクライアントが経験するハードウェア負荷の対比です。

| システムリソース | ホストマシンの負荷 | クライアントマシンの負荷 | 成功の重要要素 |
| :--- | :--- | :--- | :--- |
| **CPU使用率** | **高い** (物理同期を処理) | **中程度** (ローカルアクションを描画) | 高いシングルコア速度 |
| **システムRAM** | **高い** (プレイヤーの状態を管理) | **中程度** (標準的なゲーム負荷) | ホストには16GBを推奨 |
| **ネットワーク帯域幅** | **高い上り速度**が必要 | **低い上り速度**で可能 | 10+ Mbpsの安定した上り速度 |

これらの要求に対応するため、さまざまなマルチプレイシナリオに最適なセットアップを分類しました。
